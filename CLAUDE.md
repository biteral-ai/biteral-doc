# biteral-doc

Developer documentation site at docs.biteral.ai (EN + ES): integration guide,
REST API reference and PHP SDK reference. Docusaurus 3.9 (preset-classic,
React 19, TS config), plain npm, no Docker. Successor to `biteral-doc-old`
(Retype based); the migration happened in this repo's history, so Retype
leftovers survive (see gotchas).

## Stack and key files

- `docusaurus.config.ts`: url docs.biteral.ai, `routeBasePath: '/'` (docs ARE
  the site root), blog disabled, locales `en`/`es`, `onBrokenLinks: 'throw'`,
  navbar links (Support, GitHub, status.biteral.ai) + locale dropdown, Prism
  with php/bash/json.
- `sidebars.ts`: one autogenerated sidebar from the `docs/` tree. Order comes
  from `_category_.json` (`label`, `position`) per directory and
  `sidebar_position` in page frontmatter. Top level: Home (docs/index.md,
  `slug: /`) 1, Guide 2, PHP-SDK 3, API 4.
- `src/theme/MDXComponents.js` registers global MDX components usable in any
  .md page WITHOUT imports: `Badge`, `Button`, `Icon`, `Columns`, plus theme
  `Tabs`/`TabItem`. All .md content is compiled as MDX (JSX inside .md works).
- `src/components/`: Badge/Button/Icon/Columns (JS + css). `Icon` lazy-loads
  `src/components/icons/<name>.svg` by the `name` prop via svgr (subdirs work:
  `technology/api`, `products/naturalSearch`; a missing name renders a warning
  emoji). Badge variants are space-separable classes defined in `badge.css`:
  value, value-restriction, parameter, property, required, header, event,
  api-endpoint, http-get/post/put/patch/delete, ok, ko, warning, version,
  sdk, payload, entity, info, success, light, transparent (colors keyed to
  `--color-*` vars in custom.css). Common combos: `"property required"`,
  `"api parameter"`, `"sdk php method"`.
- `src/css/custom.css`: Infima overrides. OLD brand look, not the 2026 web
  design system: primary #0016FF (dark mode #3165ff/#235AFF), Inconsolata body
  + Inter headings from Google Fonts, 10px radii. Dark mode exists (toggle in
  navbar). Brand SVGs: `static/img/biteral-symbol*.svg`, `biteral-wordmark.svg`.

## Content layout (`docs/`)

- `index.md`: home (slug /).
- `guide/integration-data/`: intro, products, customers, `events/` (index +
  types), data-quality.
- `guide/integration-tools/`: intro + `naturalSearch.md` (the only published
  tool). `_personalizedRecommend.md`, `_profileTags.md`, `_trendAnalysis.md`
  are unpublished drafts: the underscore prefix excludes a file or directory
  from routes and sidebar.
- `api/`: connect (auth: HTTPS to api.biteral.ai with `X-API-Key` +
  `X-API-Version` headers), versions, `endpoints/` (status; products
  get/post/delete; customers get/post/delete; events post; natural-search get).
- `php-sdk/`: install (composer `biteral/biteral-sdk-php`, PHP >= 5.6,
  github.com/biteral-ai/biteral-sdk-php), examples, customers, products,
  events, entities, payloads.
- `docs/_components/`: shared MDX partials (event descriptions and property
  blurbs), imported with relative `import X from '../../../_components/...'`
  statements at the top of pages (see `guide/integration-data/events/types.md`).

Add a page: create the .md under `docs/` with `title` + `sidebar_position`
frontmatter, then create its ES twin (below). Sidebar updates automatically.

## Localization

`i18n/es/docusaurus-plugin-content-docs/current/` mirrors `docs/` 1:1,
including `_category_.json` files and `_components/` partials. RULE (readme):
every change in `docs/` must be replicated in the same file of the ES mirror.
Only doc content is translated; theme UI strings (navbar, etc.) have no
translation files and stay English in /es.

## Dev, build, deploy

- `npm install` then `npm start` (dev server, EN only). One locale per dev
  server: `npm run start -- --locale es` for Spanish.
- `npm run build` builds ALL locales into `build/` (`build/es/` for Spanish);
  `npm run serve` serves it. `npm run typecheck` for TS.
- Deploy: `.github/workflows/deploy.yml` on push to `main` builds with Node 20
  and publishes via GitHub Pages actions (upload-pages-artifact +
  deploy-pages). No CNAME file in the repo; the docs.biteral.ai binding is in
  the GitHub repo's Pages settings (not verifiable from the repo). The
  readme's `yarn deploy` / USE_SSH instructions are stale template text; the
  workflow is the real path.

## Gotchas (verified)

- `onBrokenLinks: 'throw'`: a broken internal link fails the whole build.
- Retype leftovers that Docusaurus IGNORES; do not imitate them in new pages:
  `_includes/head.html`, `index.yml` files (php-sdk/payloads, php-sdk/products)
  and the frontmatter keys `icon`, `order`, `expanded` still present on many
  pages. Only `sidebar_position`, `sidebar_label`, `title`, `slug` count.
- `src/pages/_index.tsx` is the DISABLED Docusaurus template homepage (the
  underscore excludes it); the real home is `docs/index.md`. Template leftover
  `src/pages/markdown-page.md` still publishes at /markdown-page.
- `build/` and `.docusaurus/` are generated and gitignored; `build_log.txt` is
  a committed log of a past local build. The Icon dynamic import emits a
  webpack "Critical dependency" warning at build time; harmless.
- Content links to public-site tool pages use OLD web routes, e.g.
  biteral.ai/personalized-recommend; the 2026 biteral-web route is
  /product-recommendations. Audit before republishing.
